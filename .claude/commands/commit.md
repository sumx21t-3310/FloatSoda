以下の git diff を読んで、このリポジトリのスタイルに合ったコミットメッセージを生成してください。

### ルール

- フォーマット: `<type>(<scope>): <subject>`
- type は以下から選ぶ:
  - `feat` — 新機能
  - `fix` — バグ修正
  - `refactor` — 動作を変えないリファクタリング
  - `docs` — ドキュメントのみの変更
  - `test` — テストの追加・修正
  - `chore` — ビルド・ツール・依存関係の変更
- scope は変更対象のプロジェクト名またはモジュール名（例: `RenderFlex`, `OVR`, `Widget`, `Engine`）。スコープが広い場合は省略可。
- subject は日本語で書く（例: `クリップ描画オブジェクトを追加`）
- 1行目は72文字以内に収める
- 変更の「何を」より「なぜ」を優先する
- 複数の独立した変更がある場合は、最も重要な変更を主題にして残りを本文の箇条書きで補足する
- 関連するプルリクエストがあったら番号を含めてください

### 出力形式

コミットメッセージのみを出力する（説明・前置き不要）。
本文が必要な場合は1行目の後に空行を挟んで追記する。
